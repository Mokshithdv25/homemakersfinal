# HomeMakers mobile apps (iOS & Android)

The React web app is wrapped with [Capacitor](https://capacitorjs.com/) so one codebase ships to the App Store and Google Play.

## Prerequisites

| Platform | Requirements |
|----------|----------------|
| **Both** | Node 20, npm 10, `npm ci --legacy-peer-deps` in `frontend/` |
| **iOS** | macOS, **full Xcode** (not Command Line Tools only), CocoaPods (`brew install cocoapods`), Apple Developer account for device/TestFlight |
| **Android** | Android Studio, JDK 17, Android SDK |

After cloning, if `ios/` exists but pods are missing:

```bash
cd frontend/ios/App
export LANG=en_US.UTF-8
pod install
```

## One-time setup

```bash
cd frontend
npm ci --legacy-peer-deps
```

Native projects live in `frontend/ios/` and `frontend/android/` (generated by Capacitor).

If those folders are missing:

```bash
npm run build
npx cap add ios
npx cap add android
```

## Configure the production release environment

Mobile builds bundle static files and do **not** use the dev proxy. The committed `frontend/.env.production` contains only non-secret production defaults for the Render API and canonical web URL. Put the browser-safe Supabase values in ignored `frontend/.env.production.local`:

```bash
cd frontend
# Create .env.production.local with these two values:
# REACT_APP_SUPABASE_URL=https://YOUR_PROJECT.supabase.co
# REACT_APP_SUPABASE_ANON_KEY=YOUR_ANON_OR_PUBLISHABLE_KEY
npm run verify:mobile-env
```

Never place a Supabase service-role key in any `REACT_APP_*` variable. Do not copy the local `.env.example` wholesale without removing its localhost backend URL; production validation rejects localhost and non-HTTPS origins.

Add all release callbacks to the Supabase Auth redirect allowlist:

- `https://www.homemakers.online/sign-in`
- `https://homemakers.online/sign-in`
- `in.homemakers.app://auth/callback`

The native callback uses PKCE and is handled by the app's custom URL scheme. Production currently keeps `REACT_APP_EMAIL_SIGNUP_ENABLED=false`; email/password sign-in remains available for existing accounts, but Google OAuth is the account-creation path. Do not enable email account creation until custom SMTP delivery, confirmation, resend, and password recovery have passed end-to-end tests on physical iOS and Android devices.

Google OAuth is currently exposed in the iOS shell so users are not left without an account-creation path. This does **not** make the app ready for App Store submission: implement and verify Sign in with Apple before submitting the iOS build for review.

## Build & run locally

Build the release web bundle, validate its environment, and sync it into both native shells:

```bash
npm run cap:sync
```

**iOS (Simulator or device)**

```bash
# One-time machine setup: select full Xcode, then review and accept its license.
sudo xcode-select --switch /Applications/Xcode.app/Contents/Developer
sudo xcodebuild -license
npm run cap:open:ios
# In Xcode: pick a simulator → Run (⌘R)
```

Or from CLI:

```bash
npm run cap:run:ios
```

**Android (emulator or device)**

```bash
npm run cap:open:android
# In Android Studio: Run on emulator/device
```

Or:

```bash
npm run cap:run:android
```

## Preview mobile-native UI in the browser

The app switches to a **separate mobile-native layout** (bottom tabs, mobile headers, touch-first screens) when:

- Running in the **Capacitor iOS/Android shell**, or
- You open **`http://localhost:3000/?mobile=1`** once (persists in localStorage until `?mobile=0`)

To return to desktop web: **`/?mobile=0`**

This is not responsive shrink — it is a dedicated mobile route tree under `frontend/src/mobile/`.

Point the native WebView at your dev server (same Wi‑Fi as phone/simulator):

1. Start CRA: `npm start`
2. In `capacitor.config.json`, temporarily add:

```json
"server": {
  "url": "http://YOUR_LAN_IP:3000",
  "cleartext": true
}
```

3. `npx cap sync` and run the app again.

Remove `server.url` before store builds.

## Store release checklist

1. Set the public Supabase values in ignored `.env.production.local` and run `npm run verify:mobile-env`.
2. Run `npm run cap:sync`. This performs a production mobile build with source maps disabled before Capacitor syncs the native projects.
3. Confirm `capacitor.config.json` has no temporary `server.url`, then verify the copied bundles contain neither `.map` files nor `localhost:8000`.

   ```bash
   find build ios/App/App/public android/app/src/main/assets/public -name '*.map' -print
   rg 'localhost:8000' build ios/App/App/public android/app/src/main/assets/public
   ```

   Both commands should print nothing.

4. **iOS**: Xcode → Product → Archive → Distribute to App Store Connect
   - Bundle ID: `in.homemakers.app`  
   - Version/build in Xcode project settings
5. **Android**: Android Studio → Build → Generate Signed Bundle/APK (AAB for Play Store)
   - Application ID: `in.homemakers.app`

## Icons & splash

Branded HomeMakers icons and splash screens have been generated for both native projects from `frontend/public/logo.svg` using `@capacitor/assets`:

- iOS: `ios/App/App/Assets.xcassets`
- Android: `android/app/src/main/res`

Regenerate them after any logo change, then inspect the result on light and dark devices before archiving:

```bash
cd frontend
npx capacitor-assets generate --ios --android --assetPath public --logoSplashTargetWidth 500
```

## Troubleshooting

- **Blank screen**: Run `npm run build` then `npx cap sync`. Check Safari Web Inspector (iOS) or Chrome `chrome://inspect` (Android).
- **API errors**: Run `npm run verify:mobile-env`, confirm the Render service is healthy, and ensure backend CORS allows `capacitor://localhost` / `https://localhost`.
- **Auth returns to the browser**: Confirm `in.homemakers.app://auth/callback` is in the Supabase redirect allowlist and test the installed build, not only browser mobile preview.
- **Safe area / notch**: Native shell adds `hm-native` class; fixed nav uses `env(safe-area-inset-*)`.
